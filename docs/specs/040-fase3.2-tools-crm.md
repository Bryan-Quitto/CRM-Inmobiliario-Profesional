# Fase 3.2: Integración CRM Pura (Especificación y Diseño)

## Contexto y Objetivo
Se requiere integrar capacidades para que el Bot de IA (LLM) interactúe con el CRM Inmobiliario. El objetivo es proporcionar al bot contexto histórico multi-canal de un cliente y permitirle agendar tareas (seguimientos, reuniones, visitas) de manera autónoma en el sistema.

## Decisiones Arquitectónicas (Standards)
1. **The One Trip Pattern:** Todas las consultas de lectura complejas deben consolidarse en un solo viaje a la base de datos, usando un único `.Select(...)` de LINQ to Entities (EF Core). Está estrictamente prohibido usar múltiples `await` secuenciales para cargar datos relacionados.
2. **Timezone UTC-5 (Ecuador):** Todas las fechas programadas provistas por el LLM y procesadas deben respetar el timezone del sistema (UTC-5).
3. **Validación vía JSON Schema:** La validación de campos obligatorios para herramientas de escritura (`CrearTareaCRM`) debe ser delegada al LLM mediante la configuración estricta del arreglo `"required"` en el JSON Schema de la herramienta, forzando al LLM a preguntar al usuario si falta información.
4. **Respuesta Optimizada (JSON Comprimido):** Los handlers de lectura deben devolver un JSON serializado sin formateo (minificado) para ahorrar tokens de contexto en el LLM.

## Diseño de Herramientas (AiToolDefinitions.cs)

Se deben registrar dos nuevas herramientas en `CRM_Inmobiliario.Api/Features/WhatsApp/Services/Prompts/AiToolDefinitions.cs`:

### 1. ResumirHistorialContacto
* **Descripción:** "Consulta el historial completo de un contacto en el CRM (notas, tareas, etapa de embudo y mensajes previos). Úsala cuando necesites recordar el contexto completo de un cliente o antes de transferirlo a un agente."
* **Parámetros:**
  * `searchTerm` (string, required): "Nombre completo o teléfono del contacto a buscar."

### 2. CrearTareaCRM
* **Descripción:** "Crea una tarea, recordatorio o cita en la agenda del agente en el CRM. Si el cliente pide agendar algo, DEBES pedir obligatoriamente Título, Descripción y Fecha/Hora. Si el cliente no te da alguno, PREGÚNTALE antes de usar esta herramienta."
* **Schema (Estricto):**
  ```json
  {
      "type": "object",
      "properties": {
          "titulo": { "type": "string", "description": "Título corto y claro de la tarea." },
          "descripcion": { "type": "string", "description": "Detalle completo de lo que el agente debe hacer o preparar." },
          "fechaProgramada": { "type": "string", "description": "Fecha y hora en formato ISO 8601 (ej. 2024-05-20T15:30:00). Asegúrate de pedir la hora si el cliente solo dice el día." },
          "contactoId": { "type": "string", "description": "ID del contacto (Guid), si lo conoces." },
          "propiedadId": { "type": "string", "description": "ID de la propiedad (Guid), si aplica." }
      },
      "required": ["titulo", "descripcion", "fechaProgramada"]
  }
  ```

## Cambios por Archivo

### 1. `CRM_Inmobiliario.Api/Features/WhatsApp/Tools/ResumirHistorialContacto/ResumirHistorialContactoQuery.cs`
* **Definición:** `public record ResumirHistorialContactoQuery(string SearchTerm) : IRequest<string>;`

### 2. `CRM_Inmobiliario.Api/Features/WhatsApp/Tools/ResumirHistorialContacto/ResumirHistorialContactoHandler.cs`
* **Implementación:**
  * Usar sintaxis LINQ y `let` para consolidar el historial de `WhatsappConversations` sin requerir una propiedad de navegación directa, cumpliendo el *One Trip Pattern*.
  * **Query Base:**
    ```csharp
    var query = from c in _context.Contactos
                where c.Nombre.Contains(request.SearchTerm) || c.Telefono.Contains(request.SearchTerm)
                let whatsapp = _context.WhatsappConversations
                    .OrderByDescending(w => w.UltimaActualizacion)
                    .FirstOrDefault(w => w.ContactoId == c.Id)
                select new 
                {
                    ContactId = c.Id,
                    Nombre = c.Nombre,
                    Telefono = c.Telefono,
                    EtapaEmbudo = c.EtapaEmbudo,
                    EsProspecto = c.EsProspecto,
                    EsPropietario = c.EsPropietario,
                    UltimasTareas = c.Tasks
                        .OrderByDescending(t => t.FechaInicio)
                        .Take(3)
                        .Select(t => new { t.Titulo, t.Estado, t.FechaInicio }),
                    UltimasNotas = c.Interactions
                        .OrderByDescending(i => i.FechaInteraccion)
                        .Take(3)
                        .Select(i => new { i.TipoInteraccion, i.Notas, i.FechaInteraccion }),
                    ConversacionWhatsApp = whatsapp != null ? whatsapp.HistorialJson : null
                };
    ```
  * **Retorno:** Extraer el `FirstOrDefaultAsync()` del query superior y serializar usando `JsonSerializer.Serialize` (sin identación) devolviendo un string. Si no encuentra a nadie, devolver un JSON indicando error o "Contacto no encontrado".

### 3. `CRM_Inmobiliario.Api/Features/WhatsApp/Tools/CrearTareaCRM/CrearTareaCRMCommand.cs`
* **Definición:**

  ```csharp
  public record CrearTareaCRMCommand(

  ```

### 4. `CRM_Inmobiliario.Api/Features/WhatsApp/Tools/CrearTareaCRM/CrearTareaCRMHandler.cs`
* **Implementación:**
  * Recuperar el `AgenteId` al cual asignar la tarea. Si viene `ContactoId`, consultar `c.AgenteId` del contacto (optimizando la consulta). Si no se conoce, lanzar excepción o asignar al administrador por defecto.
  * Crear la entidad `TaskItem`:
    * `Titulo = request.Titulo`
    * `Descripcion = request.Descripcion`
    * `FechaInicio = request.FechaProgramada` (El convertidor global de EF Core se encargará de guardar en UTC y recuperar en UTC-5).
    * `Estado = "Pendiente"`
    * `TipoTarea = "Programada por IA"`
    * `DuracionMinutos = 30` (por defecto).
  * Insertar mediante `_context.Tasks.Add(...)` y ejecutar `await _context.SaveChangesAsync(cancellationToken);`.
  * **Retorno:** String plano estilo `"Tarea creada exitosamente con el ID <Guid>. Se agendó para el <fecha>."` para retroalimentar al LLM.

## Consideraciones de Rendimiento y Contexto
* Limitar el `Take(3)` o `Take(5)` en las relaciones (Tareas e Interacciones) garantiza que la proyección no explote en memoria, evitando el problema del *Cartesian Explosion* en EF Core.
* La serialización JSON cruda reduce hasta un 40% del consumo de tokens al omitir saltos de línea y espacios en blanco al reinyectar el historial en el contexto del sistema.
