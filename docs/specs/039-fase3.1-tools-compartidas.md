# Fase 3.1: Herramientas Compartidas Adaptadas (Copilot y WhatsApp)

## Intent
Adapt shared tools (`BuscarPropiedades`, `ConsultarBaseConocimiento`) to be optimally consumed by the AI Copilot. Ensure prompt engineering guides the LLM to format properties predictably for UI interception (rendering rich cards) and RAG outputs are synthesized and actionable. Optimize handler payloads to save tokens when executing in the Copilot context.

## Architecture & Strategy
1. **Frontend Card Interception (Prompt Engineering)**: The AI Copilot (`AgentSystemPromptFactory`) will be instructed to ALWAYS output properties using the precise Markdown structure: `[🏠 Ver Ficha Completa: {Titulo}](/propiedades/{Id})`. The frontend chat component will detect this pattern and render a rich property card instead of plain text.
2. **RAG Output Optimization**: The Copilot will be instructed to synthesize RAG answers into actionable bullet points for the agent, avoiding raw text dumps and strict policy inventions.
3. **Payload Token Minimization**: The tools currently return verbose CSV strings or full chunks. We will adapt `BuscarPropiedadesHandler` and `ConsultarBaseConocimientoHandler` to detect if the channel is `Copilot`. If so, they will return minified, targeted JSON objects (e.g., `id`, `titulo`, `precio`, `ubicacion`) without heavy fields like `Descripcion` to aggressively reduce token usage in the LLM context.

## File-by-File Changes

### 1. `Features/AgentAi/Services/AgentSystemPromptFactory.cs`
- **Changes**: Modify `CreatePrompt()` to inject specific formatting rules and behaviors for the Copilot.
- **Rules to Add**:
  - **Propiedades**: "Cuando uses la herramienta BuscarPropiedades, DEBES presentar cada resultado usando EXACTAMENTE este formato Markdown: `[🏠 Ver Ficha Completa: {Titulo}](/propiedades/{Id})`. No uses otro formato de enlace, ya que la interfaz lo interceptará para mostrar una tarjeta visual."
  - **Base de Conocimiento**: "Cuando uses ConsultarBaseConocimiento, sintetiza la información corporativa en puntos clave o pasos accionables para el asesor. No inventes políticas."

### 2. `Features/WhatsApp/Services/Tools/BuscarPropiedadesHandler.cs`
- **Changes**: Adapt the return payload based on the `context.Channel`.
- **Logic**:
  - Check if `context.Channel == "Copilot"`.
  - If `Copilot`: Return a minified JSON array (using `System.Text.Json.JsonSerializer`) containing an anonymous object or a lightweight DTO with only `Id`, `Titulo`, `Precio`, `Sector`, `Ciudad`, `Habitaciones`, `Banos`, and `Operacion`. Exclude `Descripcion`, `UrlRemax`, and other token-heavy fields.
  - If not `Copilot` (e.g., WhatsApp): Keep returning the `FormatearCsv` method output.

### 3. `Features/WhatsApp/Services/Tools/ConsultarBaseConocimientoHandler.cs`
- **Changes**: Adapt the payload to return clean, minified context text for Copilot.
- **Logic**:
  - If `context.Channel == "Copilot"`, return a minified JSON array containing only the `Content` string of each chunk.
  - Remove verbose boilerplate text (like "Utiliza OBLIGATORIAMENTE...") from the return statement when the channel is Copilot. The system prompt in `AgentSystemPromptFactory` will already instruct the LLM on how to behave, so we don't need to waste tokens repeating instructions inside the tool's return string.
