# Fase 3.3: Herramientas Avanzadas y de Acción

## 1. Intent
Diseñar e implementar herramientas avanzadas (tools) para el Web AI Bot que permitan acciones complejas:
1. **Cotización Rápida (`GenerarCotizacionRapida`)**: Una herramienta que se integre con la Base de Conocimiento (Internal RAG) para obtener tasas financieras y plazos desde la base de datos (no hardcoded), realizar cálculos de proyección hipotecaria, y retornar una estructura JSON determinista al LLM para que la formule al usuario.
2. **Navegación Directa (`NavegacionDirecta`)**: Una herramienta de acción que permite al bot redirigir al usuario a secciones específicas del sistema (ej. vista 3D, agendar cita) emitiendo una señal de control a través del stream SSE, la cual no debe quedar persistida como texto visible en el historial de chat de la base de datos.

## 2. Architecture & Design Decisions

### 2.1 GenerarCotizacionRapida
- **Componente**: `GenerarCotizacionRapidaHandler`.
- **Flujo de Datos**:
  1. El LLM detecta la intención y llama a la tool con parámetros como: `montoPrestamo`, `enganche` (opcional).
  2. El Handler consulta el servicio de Base de Conocimiento o Repositorio Financiero para extraer: `TasaInteresActual` y `PlazosDisponibles`.
  3. Ejecuta la matemática de proyección hipotecaria (amortización estándar).
  4. Retorna al LLM un JSON estructurado con el desglose de cuotas mensuales estimadas por plazo.
- **Decisión Arquitectónica**: Las tasas y plazos nunca se queman en el prompt ni en el código (hardcoded). Siempre se inyectan en tiempo de ejecución obtenidas de la DB para evitar alucinaciones, garantizando precisión financiera.

### 2.2 NavegacionDirecta y SSE Integration
- **Componente**: `NavegacionDirectaHandler`.
- **Señal de Control**: Cuando la tool se activa, el bot inyecta una señal de control en el texto de su respuesta. Usaremos el formato especial: `[SystemAction: RedirectTo=/url]`.
- **Integración SSE**: El endpoint de Chat Streaming intercepta la respuesta. Al enviar chunks al frontend vía SSE, manda esta señal literal. El cliente SPA al parsear `[SystemAction: RedirectTo=...]` ejecuta la acción (router push) y oculta el token de la UI.
- **Prevención de Historial**: El servicio de persistencia de chat (que guarda el mensaje en la DB) debe ejecutar un filtro (Regex) para limpiar cualquier bloque `\[SystemAction: .*?\]` ANTES de guardar el texto final en la base de datos. Así, el historial queda limpio y legible al recargar el chat.

## 3. File-by-File Changes

### `src/WebAIBot/Features/Chat/Tools/GenerarCotizacionRapidaHandler.cs`
- Crear el handler que implemente la tool.
- Dependencias: `IFinancialRateRepository` (para leer tasas actuales).
- Lógica: Calcular pagos mensuales usando la fórmula de amortización.
- Retorno: Objeto C# que se serializa a JSON para el LLM.

### `src/WebAIBot/Features/Chat/Tools/NavegacionDirectaHandler.cs`
- Crear el handler que recibe el "destino".
- Mapea el destino lógico a una URL de la SPA.
- Inyecta en el contexto de respuesta el token `[SystemAction: RedirectTo=/destino]`.

### `src/WebAIBot/Features/Chat/ChatStreamingEndpoint.cs` (o `ChatHistoryService.cs`)
- Lógica de Intercepción:
  ```csharp
  string cleanMessage = Regex.Replace(rawResponse, @"\[SystemAction:.*?\]", string.Empty).Trim();
  await _chatHistoryRepository.AddAsync(cleanMessage, ...);
  ```
- **Nota**: Asegurar no romper "The One Trip Pattern" (evitar múltiples await innecesarios si se persiste el historial, agrupando las escrituras en un solo trip si es posible).
