# PROMPT PARA MEJORA DE UX: SUBIDAS MULTI-PROPIEDAD SIMULTÁNEAS

## Objetivo
Transformar el sistema de subida de imágenes (UploadContext) de un estado global único a un sistema de gestión por propiedad. Esto permitirá que el usuario pueda iniciar subidas en múltiples propiedades a la vez sin que una bloquee a la otra.

## Contexto Técnico Actual (ver Engram)
- El proyecto cuenta con un `UploadContext` que maneja el estado `isAnyUploading` de forma global.
- `PropiedadDetalle.tsx` utiliza este estado para bloquear el dropzone y mostrar el mensaje de "Procesando en segundo plano...".
- Actualmente, si una propiedad está subiendo fotos, el botón de subir se bloquea para TODAS las propiedades del CRM.

## Tareas a Realizar:

1. **Refactorizar UploadContext**:
   - Sustituir el booleano `isAnyUploading` por un objeto o Map de subidas activas: `activeUploads: Record<string, UploadProcess>`.
   - `UploadProcess` debe contener: `propiedadId`, `progreso`, `nombrePropiedad` y `estado` (cargando/completado/error).
   - Actualizar las funciones `uploadFiles` para que registren y eliminen procesos específicos en este mapa.

2. **Actualizar PropiedadDetalle.tsx**:
   - Cambiar la lógica de bloqueo: el dropzone y el botón de subir solo deben bloquearse si existe una entrada activa en el `UploadContext` que coincida con el `id` de la propiedad actual.
   - Si hay subidas en otras propiedades, esta propiedad debe permanecer habilitada para iniciar su propio proceso.

3. **Implementar Stack de Notificaciones Visuales**:
   - En el componente de visualización de carga (esquina superior derecha), realizar un `.map()` de `activeUploads`.
   - Mostrar una "pila" de mensajes animados (estilo toast) para cada proceso activo, permitiendo ver el progreso individual de cada propiedad.

4. **Optimización**:
   - Asegurar que al cerrar el modal de una propiedad, el proceso de subida continúe en segundo plano (esto ya debería funcionar, pero hay que validarlo con el nuevo mapa de estados).

---
*Nota: El contexto completo de los archivos afectados (`UploadContext.tsx`, `PropiedadDetalle.tsx`, `api/uploadImagenPropiedad.ts`) está disponible en la memoria de Engram del proyecto.*
